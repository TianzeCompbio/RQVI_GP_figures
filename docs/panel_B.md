# Panel B — RQVI Training Scalability

Training time vs. number of cells benchmark, demonstrating near-linear scalability of RQVI training.

## Design

Line plot of mean training time (minutes) vs. dataset size (thousands of cells), with shaded standard deviation bands and a linear-fit reference line. Six dataset sizes are tested (10k to 633k cells), each trained 3 times.

## Input data

| File | Description |
|------|-------------|
| `/data/tianzew/immgenT/david_final_10k_genes.h5ad` | Full dataset (633,684 cells x 10,000 genes) |
| `data/scalability_benchmark.csv` | Pre-computed benchmark results (18 rows: 6 sizes x 3 repeats) |

The benchmark CSV is generated by `scripts/benchmark_scalability.py` and contains columns: `n_cells`, `repeat`, `time_seconds`, `time_minutes`.

## Script & output

- **Benchmark script:** `scripts/benchmark_scalability.py` (generates the CSV; requires GPU)
- **Figure script:** `scripts/fig_scalability.py` (reads CSV, produces figure)
- **Output figure:** `figures/scalability.pdf`

## Key results

- Training time scales linearly with dataset size (R^2 ~ 0.9999)
- 10k cells: ~0.6 min; 633k cells: ~35.5 min
- Standard deviation is small across repeats at all sizes

## How to reproduce

```bash
# Step 1: Run benchmark (requires GPU, ~2 hours total)
python figures_version_v2/scripts/benchmark_scalability.py --device 0

# Step 2: Generate figure (CPU only)
python figures_version_v2/scripts/fig_scalability.py
```

**Note:** Step 1 requires a CUDA GPU and the RQVI model code. The pre-computed CSV is included so Step 2 can be run independently.
